# 第三部分：最小可行 Agent 互动机制

## 1. 设计思想

### 当前问题

v9.3 中，每个 Agent 独立解读因子，彼此之间没有直接互动。共识通过纯数学的加权平均计算：

```
共识 = Σ(belief_i × influenceWeight_i × confidence_i) / Σ(influenceWeight_i × confidence_i)
```

这本质是**静态聚合**，不是**动态共识形成**。

### 核心洞察

真实市场中，交易者会观察彼此的行为并调整自己的判断。这不是复杂的网络传播——即使是最简单的"我看到别人怎么看，我调整我的看法"也会产生非平凡的涌现行为。

### 设计目标

**不是**构造真实的社交网络，**而是**让 Agent 的信念在互动中**动态演化**，从而：

1. 展示"初始分歧 → 互动 → 共识（或分歧加大）"的动态过程
2. 产生"共识脆弱性"——某些条件下互动导致共识，另一些条件下互动导致更大分歧
3. 可视化"信息异质性如何影响共识形成"

### 设计约束

- 纯数学，零 LLM 调用
- 5行核心公式
- 不修改已有 v9 代码
- 可独立开关（不启用互动层 = v9.3 原始行为）

## 2. 数学表达

### 核心公式：社交信念更新

```
b_i^{(t+1)} = (1 - α_i) · b_i^{(t)} + α_i · b̄_visible_i^{(t)}
```

其中：
- `b_i^{(t)}` = Agent i 在第 t 轮的信念
- `α_i` = Agent i 的**社交开放度**（0 = 完全不理会别人，1 = 完全从众）
- `b̄_visible_i^{(t)}` = Agent i **能看到的**其他 Agent 的加权平均信念

### 可见性约束（关键创新）

**Agent 不是看到所有其他 Agent。**每个 Agent 只能看到与它有**因子重叠**的其他 Agent：

```
visible_i = { j | factors_i ∩ factors_j ≠ ∅, j ≠ i }
```

这意味着：
- 🏦 Institution（liquidity+policy+fundamental）能看到 💎 Value（fundamental）、🤖 Quant（liquidity+fundamental）、😱 Panic（liquidity）、🏛️ PolicyAgent（policy+liquidity）
- 🏦 Institution **看不到** 🏄 Trend（narrative）、📡 Media（narrative+policy）、🦉 Contrarian（narrative）、🐜 Retail（narrative）
- 🏄 Trend（narrative）只能看到 📡 Media（narrative+policy）、🦉 Contrarian（narrative）、🐜 Retail（narrative）

**这天然产生信息茧房。**不同 Agent 看到不同"别人"的观点——这是异质性的第二层。

### 可见信念的加权平均

```
b̄_visible_i = Σ_{j ∈ visible_i} (b_j · w_j · conf_j) / Σ_{j ∈ visible_i} (w_j · conf_j)
```

### 社交开放度 α_i 的设计

不同 Agent 对他人观点的敏感度不同：

| Agent | α | 解释 |
|-------|---|------|
| 🏦 Institution | 0.15 | 机构有独立研究，不易动摇 |
| 💎 Value | 0.05 | 价值投资者最独立 |
| 🏄 Trend | 0.50 | 趋势交易者高度关注他人行为 |
| 😱 Panic | 0.70 | 恐慌者最易受他人影响 |
| 🤖 Quant | 0.10 | 量化模型不受情绪影响 |
| 📡 Media | 0.45 | 媒体天然放大和跟随 |
| 🦉 Contrarian | -0.30 | **负的**——看到别人看空，反而看多 |
| 🐜 Retail | 0.60 | 散户高度从众 |
| 🏛️ PolicyAgent | 0.20 | 政策制定者相对独立 |

**注意：**Contrarian 的 α = -0.30。他看到别人看空时，反而调整看多。这使得互动可能产生反直觉的动态。

### 收敛条件

互动在多轮中迭代，直到满足以下任一条件：
1. **收敛**：所有 Agent 信念变化 < 2（已稳定）
2. **最大轮次**：达到 10 轮（防止无限循环）
3. **发散检测**：belief_std 连续 3 轮增长（分歧在加大——不是收敛而是极化）

## 3. 数据结构

```typescript
// === 新增类型 (v9.5/types.ts) ===

/** Agent 社交参数 */
interface SocialProfile {
  alpha: number;           // 社交开放度 (-1~1)
  visibleAgentIds: string[]; // 能看到的其他 Agent (由因子重叠决定)
}

/** 单轮互动状态 */
interface InteractionRound {
  round: number;
  beliefs: Record<string, number>;      // 每个 Agent 的信念
  beliefChanges: Record<string, number>; // 每个 Agent 的信念变化
  meanBelief: number;
  beliefStd: number;
  converged: boolean;
}

/** 互动结果 */
interface InteractionResult {
  rounds: InteractionRound[];
  finalBeliefs: Record<string, number>;
  totalRounds: number;
  convergenceType: "converged" | "max_rounds" | "diverged";
  // 关键指标
  beliefShift: Record<string, number>;  // 最终信念 - 初始信念
  consensusFormed: boolean;              // 是否形成了共识
  polarizationIncreased: boolean;        // 分歧是否加大了
}

/** 共识度量指标 */
interface ConsensusMetrics {
  consensusScore: number;      // 0-100, 共识强度
  polarizationScore: number;   // 0-100, 极化程度
  fragilityScore: number;      // 0-100, 共识脆弱性
}
```

3. **反事实分析**：展示系统的脆弱性和关键依赖
4. **叙事转型**：从"预测结果是什么"变成"共识是如何形成的"

# 第四部分：三个新指标

## 指标 1: Consensus Score（共识强度）

### 定义

Consensus Score 衡量一群 Agent 在多大程度上形成了**方向一致的共识**。它不是"预测准确性"的指标——它是"这群Agent是否达成了一致"的指标。

### 计算方法

```
Consensus Score = f(kuramoto_r, |consensus|, 1 - belief_std/100)

其中:
  kuramoto_r     ∈ [0, 1]   — 相位同步度 (来自物理学)
  |consensus|    ∈ [0, 100] — 共识的绝对值 (方向强度)
  belief_std     ∈ [0, 100] — 信念标准差 (异质性)

组合公式:
  sync_component     = kuramoto_r × 100
  direction_component = min(|consensus|, 50) × 2    // 规范化到 [0, 100]
  agreement_component = max(0, 100 - belief_std)     // 低分歧 = 高共识

  Consensus Score = 0.4 × sync_component + 0.3 × direction_component + 0.3 × agreement_component

  → 取值范围: [0, 100]
    - 0-30:   弱共识（Agent各说各话）
    - 30-60:  中等共识
    - 60-80:  强共识
    - 80-100: 极强共识（所有人几乎一致）
```

### 展示方式

一个圆形仪表盘，指针从0到100。颜色从红（0）→ 黄（50）→ 绿（100）。

### 可视化建议

- **径向进度条**：类似速度表，有视觉冲击力
- **动画**：共识从初始值到最终值的变化过程（Tween动画）
- **分区着色**：红区(0-30)、黄区(30-60)、绿区(60-100)

---

## 指标 2: Polarization Score（极化程度）

### 定义

Polarization Score 衡量 Agent 群体**分裂成对立阵营**的程度。不是"分歧大不大"（那是 belief_std），而是"是否形成了两个极端对立的阵营"。

### 计算方法

```
Polarization Score = g(bullish_strength × bearish_strength, belief_std, neutral_ratio)

其中:
  bullish_strength = Σ(b_i × w_i) / Σw_i  for b_i > 15
  bearish_strength = |Σ(b_i × w_i)| / Σw_i  for b_i < -15
  neutral_ratio    = count(|b_i| ≤ 15) / N

  // 极化的本质: 两端都强 + 中间少
  extremity_product = bullish_strength × bearish_strength / 2500  // 规范化
  bimodality        = (1 - neutral_ratio) × (belief_std / 100)
  
  Polarization Score = (0.5 × extremity_product + 0.5 × bimodality) × 100

  → 取值范围: [0, 100]
    - 0-30:   低极化（共识型或混乱型）
    - 30-60:  中度极化
    - 60-80:  高度极化（两个对立的叙事阵营）
    - 80-100: 极端极化（市场即将"选方向"）
```

### 关键洞察

**Polarization ≠ Disagreement。**

- 高 disagreement + 低 polarization = 各说各话，没有形成阵营
- 高 disagreement + 高 polarization = 两个阵营激烈对抗
- 低 disagreement + 低 polarization = 温和共识

Polarization 是**更有趣**的指标，因为它暗示了"市场即将选方向"的临界状态。

### 展示方式

两个对立的箭头，向外拉扯。中间显示分数。极度极化时箭头剧烈抖动。

### 可视化建议

- **双峰直方图**：展示 Agent 信念分布，两条峰代表两个阵营
- **拉扯动画**：两个箭头向外拉，力度与 Polarization Score 成正比
- **阵营颜色**：绿色（多头阵营）vs 红色（空头阵营），中间灰色（中立）

---

## 指标 3: Consensus Fragility Score（共识脆弱性）

### 定义

Consensus Fragility Score 衡量**当前共识有多容易被打破**。一个高共识但极高脆弱的系统意味着：表面上一群人同意，实际上只要一个关键 Agent 改变主意，共识就会翻转。

### 计算方法

```
Fragility Score = h(agents_to_flip, max_marginal_impact, blindness_effect)

其中:
  agents_to_flip: 从 diagnostics.counterfactuals 中获取
    — 需要移除多少 Agent 才能翻转共识方向

  max_marginal_impact: 从 diagnostics.attribution 中获取
    — 单一 Agent 的最大贡献百分比
    — 如果某个 Agent 贡献 > 40%，脆弱性极高

  blindness_effect: 从 diagnostics.counterfactuals 中获取
    — 关闭信息盲区后共识偏移的绝对值
    — |Δconsensus| 当 blindnessDisabled

  // 三个因素组合
  concentration_risk = max(0, (max_marginal_impact - 25) / 75)  // 贡献>25%开始计入
  flip_risk          = max(0, 1 - agents_to_flip / agents_count) // 少=脆弱
  blindness_risk     = min(blindness_effect / 30, 1)             // 偏移>30=极度脆弱

  Fragility Score = (0.4 × concentration_risk + 0.3 × flip_risk + 0.3 × blindness_risk) × 100

  → 取值范围: [0, 100]
    - 0-25:   稳健（共识有多重支撑）
    - 25-50:  中等脆弱
    - 50-75:  脆弱（少数Agent的移除会改变共识）
    - 75-100: 极度脆弱（共识依赖单一关键Agent）
```

### 关键洞察

这个指标直接对应到**真实市场的脆弱性**：

- 2008年金融危机前：市场共识是"房价永远涨"，但这个共识极度脆弱——它依赖"信用评级不会集体下调"这一个假设
- 2020年新冠崩盘：市场共识是"牛市继续"，但这个共识极度脆弱——一个黑天鹅事件就让它瓦解

**Fragility Score 是三个指标中最有学术价值的一个。**

### 展示方式

一座积木塔/多米诺骨牌。脆弱性越高，塔越摇摇欲坠。点击可以看到"哪些 Agent 是关键的"。

### 可视化建议

- **积木塔动画**：脆弱的共识 = 抽掉一块积木塔就倒；稳健的共识 = 抽掉几块还能稳
- **多米诺骨牌**：展示"如果X改变主意，Y也会跟着改变"的连锁反应
- **关键依赖图**：网络图，中心节点是最关键的Agent，节点大小 = 边际贡献

## 三指标联合解读

三个指标组合起来，给出市场的"共识状态画像"：

| Consensus | Polarization | Fragility | 状态解读 |
|-----------|-------------|-----------|---------|
| 高 (>60) | 低 (<30) | 低 (<30) | **稳健共识** — 所有人基于相似信息得出相似结论 |
| 高 (>60) | 低 (<30) | 高 (>60) | **脆弱的共识** — 表面一致但依赖关键Agent |
| 高 (>60) | 高 (>60) | 高 (>60) | ⚠️ **不可能三角** — 如果polarization高，consensus不可能高。检查计算。 |
| 低 (<30) | 高 (>60) | 高 (>60) | **两极对抗** — 两个阵营激烈博弈，共识随时可能翻转 |
| 低 (<30) | 低 (<30) | 低 (<30) | **认知迷雾** — Agent各说各话，但也没形成阵营 |
| 低 (<30) | 高 (>60) | 低 (<50) | **健康分歧** — 有对立但系统稳健，分歧是结构性的 |

# 第五部分：项目叙事重新设计

## 1分钟介绍（电梯演讲）

> 金融市场中，8个不同的人看到同一条新闻，会得出8种不同的结论。
>
> 有人看到风险，有人看到机会，有人跟随大众，有人逆势而为。
>
> **他们的共识是如何形成的？这个共识有多脆弱？**
>
> SwarmAlpha 不是预测市场的工具——它是一个研究"市场共识如何从信息碎片中涌现"的实验平台。
>
> 我建造了一个由8个AI Agent组成的微型金融市场。每个Agent有不同的信息获取权限、不同的思维方式、不同的社交倾向。当一条新闻进入这个系统，你可以看到：分歧如何产生，共识何时形成，以及——什么条件下共识会瞬间瓦解。
>
> 在60个历史事件的回溯中，这个系统展示了一个令人不安的事实：**大多数市场共识都是脆弱的**——只需要移除1-2个关键参与者，共识方向就会翻转。
>
> 我是贺孟元，一个高一学生。我用两个月和AI工具建造了这个实验室。

## 3分钟路演版本

### 开场（30秒）—— 一个真实的问题

2020年3月，新冠疫情全球爆发。美股10天4次熔断。同一条新闻——"新冠疫情爆发"——在不同人眼里是完全不同的故事。

价值投资者看到：RSI跌到15，历史胜率78%，这是买入机会。
趋势交易者看到：均线全面死叉，不要接飞刀。
恐慌者看到：所有人都在卖，我也要卖。
逆向投资者看到：极度恐慌 = 历史性买入机会。

**问题来了：这些完全不同、甚至互相矛盾的认知，最终是如何形成"市场价格"这一个数字的？**

### 我的方案（60秒）—— 一个可观测的微型市场

我建造了 SwarmAlpha——一个由8个AI Agent组成的微型金融市场实验室。

每个Agent有不同的"信息盲区"：
- 🏦 机构投资者看到流动性、政策和基本面——但看不到市场叙事
- 🏄 趋势交易者只看到叙事——对基本面一无所知
- 💎 价值投资者只关注基本面——完全不知道市场在恐慌

56%的Agent对之间**没有任何共享的方向性信息**。这模拟了真实市场中的"信息异质性"——不同参与者基于完全不同的信息子集做出判断。

然后这些Agent不是孤立决策。他们可以观察彼此的观点，调整自己的信念。我会追踪这个动态过程。

### 我发现的三件事（60秒）

**第一，共识的形成是非线性的。** 影响力加权可以制衡空头联盟——在我的实验中，加入机构投资者的影响力权重后，Up准确率从14%提升到43%。

**第二，大多数共识是脆弱的。** 我在系统中运行了反事实分析：如果移除某个关键Agent，共识方向会变吗？在60个历史事件中，多数情况下仅需移除1-2个Agent就能翻转共识方向。

**第三，信息盲区不是bug——它是异质性的来源。** 当所有Agent看到相同信息时，信念差异只有17.9。当强制信息盲区后，差异扩大到58.5——这才是真实市场中"一千个人有一千个哈姆雷特"的数学表达。

### 结尾（30秒）—— 为什么这重要

SwarmAlpha 不预测市场涨跌。它的价值在于：**让不可见的"共识形成过程"变得可见、可测量、可实验。**

金融危机、泡沫形成、恐慌踩踏——这些本质上都是"共识脆弱性"的体现。通过建造这个微型市场，我试图理解：**在信息异质的条件下，共识何时稳健？何时脆弱？何时会瓦解？**

## 5分钟答辩版本

### Part 1: 研究问题 (60秒)

我的核心研究问题是：

**"在信息异质条件下，多个智能体如何形成市场共识？"**

这个问题有三个子问题：
1. **认知差异的来源**：Agent之间的观点差异，是来自不同的个性（Personality），还是来自不同的信息获取（Information）？
2. **共识形成的机制**：从分散的个体信念到统一的群体共识，中间发生了什么？
3. **共识的脆弱性**：什么条件下，看似稳固的共识会瞬间瓦解？

### Part 2: 方法设计 (90秒)

我设计了四个层次的实验控制：

**第一层：正交因子提取。** 不让LLM直接判断涨跌——而是让它从新闻中提取5个正交因子（流动性、政策、基本面、叙事、不确定性）。每个因子在数学上是独立的。这避免了"让LLM预测市场"的黑箱问题——LLM只做信息结构化。

**第二层：强制信息盲区。** 8个Agent每个只能看到1-3个方向因子。56%的Agent对之间没有任何共享的方向性信息。这让我可以区分"个性差异"和"信息差异"——我的实验证明，后者比前者重要3倍（信念差异 175 vs 60）。

**第三层：异质解释函数。** 即使两个Agent看到相同的因子，他们也会得出不同的结论——因为他们的"解释风格"不同。恐慌者用S型放大函数，逆向者用反转函数，价值投资者用逆势函数。

**第四层：社交互动。** Agent可以观察与自己共享因子的其他Agent的观点，并根据自己的社交开放度调整信念。逆向者的社交开放度是负的——看到别人恐慌，他反而更坚定。

### Part 3: 关键发现 (90秒)

**发现1：信息 > 个性。** 在v5实验中，纯Personality Prompt产生的Agent信念差异只有60点。引入信息不对称（不同Agent看到不同信息片段）后，差异扩大到175点。这证明了：市场中的观点分歧，主要来自信息异质性，而不是个性差异。

**发现2：线性共识聚合是硬天花板。** 共识计算的本质是加权平均——它不能创造信息，只能重新分配。在我的反投票测试中，关闭影响力加权后，共识与纯投票的差异仅6.7-7.6点。这意味着任何基于线性聚合的系统，其表现上限受限于输入信息的质量。

**发现3：共识脆弱性是系统性的。** 在60个历史事件的反事实分析中，大多数共识方向仅需移除1-2个关键Agent就能翻转。这个发现与真实市场高度吻合——2008年的"房价永远涨"共识、2020年3月的"牛市继续"共识，都是在少数关键假设被打破后瞬间瓦解的。

**发现4：盲区产生真正的异质性。** 当信息盲区开启时，Agent信念标准差是58.5。当盲区关闭（所有Agent看到全部信息）时，标准差降至17.9。40点的振幅证明了信息盲区是异质性的主要驱动力。

### Part 4: 局限性与未来 (60秒)

我诚实地报告系统的局限性：

- **准确率天花板**：最佳版本51.7%，低于"永远猜涨"基线60%。但这不重要——因为系统不是预测工具。
- **事件库规模**：60个事件，统计推断的置信区间仍然较宽。
- **时间维度缺失**：只判断方向，不判断时点和幅度。
- **单向度市场**：只模拟单一资产，无跨资产反馈。

未来方向：
- **共识脆弱性的预测能力**：Fragility Score 能否作为市场风险的早期预警？
- **跨资产共识传播**：股市的共识如何传播到债市、汇市？
- **Agent自主信息获取**：从被动接收信息到主动搜索、筛选、验证。

长回答：
- 我没有声称这些Agent'像真人'。我声称的是：当8个Agent被赋予不同的信息获取权限和不同的决策函数时，他们会产生**结构性的认知差异**——这些差异的**模式**（而不是绝对值）与真实市场中的信息不对称有结构上的同构性。
- 在v5实验中，我做了对照：Personality Prompt（让LLM扮演不同性格）产生的Agent差异只有60点；Information Blinding（给不同Agent不同信息片段）产生的差异是175点。这证明**信息不对称 > 个性差异**——这个发现对真实市场也是有效的，因为它是一个结构性结论，不依赖Agent是否'像人'。
- 所有模型的本质都是简化。问题是：这个简化保留了关键的结构，还是丢失了它？我认为SwarmAlpha保留了市场共识形成的**三个关键结构**：信息异质性、认知差异、社交影响。"

### Q5: "你用的是大语言模型，LLM本身就有偏见。你怎么保证你的结果不是LLM偏见的产物？"

**最佳回答：**

"这是一个我认真对待的问题。我的应对有三层：

**第一层：模板模式对照。** 系统有两种运行模式——LLM模式（用DeepSeek提取因子）和模板模式（用关键词正则表达式提取因子）。所有实验都在两种模式下运行并对照。如果LLM偏见主导了结果，两种模式的结果应该有系统性差异——但实际差异主要在幅度而非方向。

**第二层：因子提取而非判断。** v9的关键架构决策是：不让LLM判断'涨还是跌'，只让它提取5个正交因子的数值。LLM做结构化，不做判断。Agent的信念是纯数学计算的结果。这最大程度隔离了LLM偏见。

**第三层：公开LLM偏见。** DeepSeek确实系统性偏空（即使给Bull persona也倾向输出负值）。我在PROJECT_EVALUATION中诚实记录了这一点。这本身是一个有趣的研究发现——**AI的认知偏见如何影响群体决策？** —这正是我的研究问题的一个维度。"

| 已有功能回归 | 极低 (2%) | 高 | 不修改核心引擎，新模块可独立开关 |
