## 一、项目概述

### 这是什么？

SwarmAlpha 是一个 **Agent-Based Market Simulation（ABMS）实验平台**。输入一条金融新闻，8 个拥有不同决策框架、资本权重和信息盲区的 AI Agent 通过影响力网络、信念扩散、资金流模拟，涌现出市场共识。

**它不是预测工具**——最诚实的回测显示准确率 51.7%（永远猜涨基线 60%）。它是一个用于理解"市场共识如何从异质参与者互动中形成"的实验平台。

### 灵感来源

索罗斯的**反思性理论（Reflexivity）**——市场参与者的认知会反过来影响市场本身。这个项目试图用 AI Agent 来模拟这个反馈循环。

### 一句话

> 不让 LLM 判断涨跌——让它提取正交市场因子，然后让 8 个不同盲区的 Agent 各自解读，看看共识如何涌现。

## 二、核心工作流

```
用户输入: "2020年3月，新冠疫情全球爆发，美股四次熔断..."

  ┌─ Step 1: 因子提取 (1次LLM 或 模板) ─────────────────────┐
  │  LLM 提取 5 个严格正交因子:                               │

  │  Narrative: +70   Uncertainty: 95                         │
  └──────────────────────────────────────────────────────────┘
                              │
  ┌─ Step 2: Agent 解读 (8+1 Agents, 纯数学) ────────────────┐
  │  每个 Agent 只看到部分因子 (强制信息盲区):                  │
  │                                                            │
  │  🏦 Institution → [liquidity, policy, fundamental]        │
  │     "政策兜底+流动性危机 → 短期恐慌，中期看政策"  信念:+15  │
  │  💎 Value       → [fundamental]                            │
  │     "RSI<20 = 历史胜率78%，恐慌折价显著"          信念:+65  │
  │  🏄 Trend       → [narrative]                              │
  │     "趋势明确向下，均线死叉，不接飞刀"            信念:-70  │
  │  😱 Panic       → [liquidity]                              │
  │     "流动性枯竭！所有人都在卖！"                  信念:-90  │
  │  🤖 Quant       → [liquidity, fundamental]                │
  │     "RSI<25+VIX>35 = 强买入信号(胜率72%)"        信念:+45  │
  │  📡 Media       → [narrative, policy]                      │
  │     "崩盘叙事加速传播，但饱和度已近极值"          信念:-55  │
  │  🦉 Contrarian  → [narrative] (负权重)                     │
  │     "共识极度看空——历史上反转概率升高"           信念:+50  │
  │  🐜 Retail      → [narrative]                              │
  │     "社交媒体都在说崩了...我也好慌"               信念:-40  │
  │  🏛️ PolicyAgent → [policy, liquidity] (零资本，仅影响共识) │
  └──────────────────────────────────────────────────────────┘
                              │
  ┌─ Step 3: 共识涌现 (3轮, 纯数学) ─────────────────────────┐
  │  影响力网络扩散 → 信念更新 → 资金流 → 价格形成             │
  │  共识: -8 (中性偏空)                                       │
  │  资金流: -18.5           价格Δ: -1.8%                      │
  │  Regime: SYSTEMIC_COLLAPSE                                 │
  │  涌现: PANIC_SELLING + CONTRARIAN_SURGE                    │
  └──────────────────────────────────────────────────────────┘
```

## 六、当前遇到的困难

### 🔴 致命困难

#### 1. 永远猜涨基线 60% — 超越它极其困难

S&P 500 历史上约 60% 的时间在上涨。任何预测系统如果准确率低于 60%，实际不如"永远猜涨"策略。当前最佳 v9.3（仅Rule1 LLM）= 51.7%，仍低于基线 8.3pp。

v8.1 聚类+动态K 曾达到 71.7%（首次超越基线），但那是模板模式（0次LLM，确定性），在 LLM 模式下尚未复现。

#### 2. 线性共识聚合是硬天花板

共识计算的本质是加权平均：

```
共识 = Σ(belief_i × influence_i × confidence_i) / Σ(influence_i × confidence_i)
```

线性组合的数学性质：**不能创造信息，只能重新分配**。反投票测试证实了这一点——关闭影响力加权后，共识与纯投票的差异仅 6.7-7.6pt（3个版本仅提升 0.9pt）。

v8 的非线性共识（聚类/幂律/熵权）将 vs线性差异推至 16.5pt，但引入了新的不稳定性（蒙特卡洛翻转率 11.6%）。

#### 3. 信息论上限 ~59-62%

```
信息提取上限: ~53% (9维特征, 事件分类器V2基准)
+ 反身性增益: ~+4pp (叙事冲击创造新信息)
+ 异质决策增益: ~+2pp (不同函数形态提取不同信号)
─────────────────
理论上限:       ~59-62%
```

当前已达到信息上限的 ~94%。超过 60% 需要的不是调参，是**架构突破**。

### 🟡 重要困难

#### 4. Down 事件与 Up 事件的预测能力极度不对称

| 版本 | Up 准确率 | Down 准确率 | 不对称程度 |
|------|----------|------------|-----------|
| v4.1 混合 | 100% | 20% | 5:1 |
| v6 加权 | 43% | 60% | 1:1.4 |
| v9.2 Hybrid | 50% | 71% | 1:1.4 |
| v9.3 仅Rule1 LLM | 39% | **100%** | 1:2.6 ⚠️ |

没有一个版本能同时在 Up 和 Down 上都表现好。v9.3 牺牲 Up（39%）换 Down（100%），v9.2 提升 Up（50%）但 Down 降至 71%。这是系统级的 trade-off，至今未解决。

#### 5. V 型反弹 vs L 型衰退 — 无法可靠区分

系统的超卖看多偏差在真正下跌时是毒药。当 RSI<15、VIX>35 时，校准层强制看多——这在 V 型反弹中正确（如 2008 LTCM、2016 英国脱欧），但在 L 型衰退中致命（如 2008 雷曼、2015 A股股灾、2020 新冠首爆）。

**核心矛盾**：区分"超卖=买入"和"超卖=还有更多下跌"需要结构损伤评估（杠杆/信贷利差/偿付能力），但这些数据无法从新闻文本中可靠提取。

#### 6. LLM 非确定性 — 同一输入不同输出

真实 LLM 回测中，同一事件两次运行可能给出不同的共识方向和准确率。v5 用 168 次真实 LLM 调用仅得 36%，而 v6 的确定性数学引擎（0次LLM）反而达到 50%。

v9 的因子提取策略缓解了这个问题（让 LLM 只提取因子值而不判断涨跌），但 LLM 的因子值仍存在 ±15-20 的波动。

### 🟢 次要困难

#### 7. 事件库规模不足

60 个历史事件（36 up / 17 down / 7 neutral），每个方向的样本量仍不足以做统计推断。需要扩充至 ~200 事件才能得到 95% 置信区间 < ±5pp。

#### 8. 时间维度缺失

当前只预测方向（up/down/neutral），不预测：
- 反弹时点（2020 新冠：2 月 24 日 vs 3 月 23 日真正的底）
- 持有期（什么时候方向会反转）
- 幅度（涨/跌多少）

#### 9. 无真实资本约束

资金流计算是 `belief × capitalWeight × confidence`——抽象数字。真实市场有头寸限制、流动性约束、监管要求、异质时间偏好，这些都没有建模。

#### 10. 无跨资产反馈

系统只模拟单一市场（S&P 500 方向）。真实恐慌中资金会在股市/债市/汇市/商品间流动（flight to quality / carry trade unwind / risk-off selloff）。Yahoo Finance 数据已接入但仅作为静态背景，不参与动态反馈。

#### 11. Agent 无自主信息获取

Agent 被动接收预计算的 WorldModel 字段。真实 Agent 会主动搜索、交叉验证、判断信息可靠性、选择性忽略或放大特定信息。

#### 12. 确定性数学 → 不可复现涌现

v6/v7/v9（模板模式）的确定性数学意味着：相同输入=相同输出。真正的涌现系统应有不可预测性——相同初始条件每次运行产生微妙不同的结果。当前仅在 Agent Observe() 注入 ±5% 观测噪声。

## 九、快速开始

### 环境要求

- Node.js ≥ 18
- npm ≥ 9
- （可选）DeepSeek API Key（用于 LLM 模式）

### 安装与启动

```bash
# 1. 克隆项目
git clone <repo-url>
cd swarmalpha

# 2. 安装依赖
npm install

# 3. 配置环境变量（可选）
cp .env.local.example .env.local
# 编辑 .env.local，填入 DEEPSEEK_API_KEY

# 4. 启动开发服务器
npm run dev
# 打开 http://localhost:3000
```

### API 调用

```bash
# v9 正交五因子模式（推荐，1次LLM调用）
curl -X POST http://localhost:3000/api/swarm \
  -H "Content-Type: application/json" \
  -d '{
    "version": "v9",
    "news": "美联储宣布紧急降息50个基点，超出市场预期",
    "rounds": 3,
    "llmConfig": {"provider": "deepseek", "model": "deepseek-chat"}
  }'
```

### 运行回测

```bash
# 60事件模板回测
npx tsx test/expanded-backtest.ts

# v8.5 科学验证（样本外 + 蒙特卡洛）
npx tsx test/v8.5-validation.ts

# v9.3 Neutral 消融实验
npx tsx test/v9.3-neutral-ablation.ts --llm  # 需要 DeepSeek API Key
```

## 十、项目定位与诚实判定

### SwarmAlpha 是什么？

```
预测工具？         → ❌ 不是。51.7%准确率 < 60%永远猜涨基线。
AI 金融产品？      → ❌ 不是。没有交易信号，没有风险管理，没有付费用户。
Agent-Based Model？ → ✅ 是。一个诚实的 ABM 实验平台。
```

### 它的价值不在于预测市场

1. **证明了一条技术路径**：从简单投票 → 信息不对称 → 异质决策 → 正交因子 → 非对称共识的演化路线是可行的
2. **积累了一套验证方法**：信息不对称测试、反投票测试、涌现行为检测、消融实验框架
3. **建立了一种实验文化**：诚实回测 > 虚高数字、实验驱动 > 调参刷榜、删除无效代码 > 保留"可能有用"的代码
4. **提出了正确的问题**：Personality vs Information、Linear vs Non-linear、Blind Spot vs Full Information——这些问题的价值超越了金融预测

### 如果继续，最值得突破的三个方向

1. **非线性共识聚合 v2**：自适应修剪 + 动态权重，目标 vs线性差异 >15pt
2. **Agent 自主信息获取**：从被动接收 → 主动搜索、交叉验证、可靠性判断
3. **随机 Agent 生命周期**：注入 Agent 失败/退出/新进入，创造真正的涌现不可预测性
